--1.JVM架构
	Java 虚拟机（JVM）从整体架构上通常被划分为 五大核心组件。这五个部分协同工作，完成字节码的加载、执行、内存管理等任务

	**类加载子系统，负责将.class 字节码文件加载到内存中，并转换为 JVM 可识别的运行时数据结构
		主要流程：   加载（Loading）：查找并读取 .class 文件
					链接（Linking）：验证、准备（分配静态变量内存）、解析（符号引用转直接引用）
					初始化（Initialization）：执行静态代码块和静态变量赋值
	**运行时数据区(JVM内存模型)：分为以下五个区域
		方法区：线程共享，存储类信息、常量、静态变量、JIT 编译后的代码（JDK 8+ 用 元空间 Metaspace 替代永久代）
		堆：线程共享，存放所有对象实例和数组，是 GC 的主要区域（分新生代、老年代）
		虚拟机栈：线程私有，每个线程私有，存储方法调用的栈帧（含局部变量表、操作数栈等）
		本地方法栈：线程私有，为 native 方法（如 C/C++）服务
		程序计数器：线程私有，记录当前线程正在执行的字节码指令地址，是唯一不会发生 OutOfMemoryError 的区域
	**执行引擎：
		作用：将字节码翻译成底层系统指令并执行。
		核心组件：
		    解释器（Interpreter）：逐条解释执行字节码，启动快但效率低
		    即时编译器（JIT Compiler）：将热点代码编译为本地机器码，提升性能
		    垃圾回收器（Garbage Collector）：自动回收堆中无用对象（虽然 GC 逻辑常归入运行时数据区管理，但执行由引擎驱动）
	**本地方法接口（JNI）：
		作用：提供 Java 代码调用 本地语言（如 C/C++） 编写的函数的能力。
		通过 native 关键字声明方法，由 JNI 桥接 JVM 与本地库。
	**本地方法库（Native Method Libraries）
		作用：包含用 C/C++ 等语言编写的本地方法实现（如 System.currentTimeMillis() 底层调用操作系统 API）
