--1.进程与线程
	IO:input/output，指计算机系统与外部环境之间进行数据交换的过程。
	进程：通常我们的程序是由指令和数据组成的，程序运行就需要把指令加载到CPU，数据加载至内存，而进程就是用来加载指令，管理内存，管理IO的。当一个程序被运行，从磁盘加载这个程序的代码至内存，就开启了一个进程。
	线程：可以看成是指令流，将指令流中的一条条指令以一定的顺序交给CPU执行。一个进程内可以有多个线程。

	二者对比：
	**从个体上看：进程基本上是相互独立的，线程是进程的一个子集（进程拥有共享资源，比如说内存空间供内部的线程共享）
	**从通信上看：进程间的通信较为复杂，同一台计算机的进程通信成为IPC，不同计算机间进程通信需要遵守共同的协议，如HTTP
				 线程通信较为简单，因为其共享进程内的资源，比如多个线程可以访问同一个共享变量

--2.并发与并行
	单核 cpu 下，线程实际还是 串行执行 的。操作系统中有一个组件叫做任务调度器，将 cpu 的时间片（windows下时间片最小约为 15 毫秒）分给不同的程序使用，只是由于 cpu 在线程间（时间片很短）的切换非常快，人类感
		觉是 同时运行的 。总结为一句话就是： 微观串行，宏观并行 
	并发：像线程轮询使用CPU的做法就叫并发，强调同一时间应对多件事的能力
	并行：多核cpu下，每个核 都可以调度运行线程，这时候线程可以是并行的，强调同一时间做多件事的能力

--3.异步同步
	通常以调用方式区分，需要等待结果返回再继续运行的是同步，反之是异步

--4.java创建线程的几种方式
	项目中最常用的是线程池，可以线程复用，提升性能，也可以支持定时等调度任务；
	另外比较推荐的可以通过实现runnable接口实现，重写run()；
	如果需要返回结果可实现Callable 接口 + FutureTask,如下：

		class MyCallable implements Callable<String> {
		    @Override
		    public String call() throws Exception {
		        return "任务完成，返回结果";
		    }
		}

		// 使用
		FutureTask<String> future = new FutureTask<>(new MyCallable());
		new Thread(future).start();

		try {
		    String result = future.get(); // 阻塞等待结果
		    System.out.println(result);
		} catch (Exception e) {
		    e.printStackTrace();
		}

--5.线程运行原理
	**栈与栈帧
		JVM内存模型中的栈内存就是给线程用的，每个线程启动后虚拟机就会为其分配一块栈内存；
		每个栈由多个栈帧（Frame）组成，对应着每次方法调用时所占用的内存；
		每个线程只有一个活动栈帧，对应着当前正在执行的方法
	**线程上下文切换（Thread Context Switch）
		当由于某些原因CPU不再执行当前的线程，转而执行另一个线程的代码。（线程的CPU时间片用完，垃圾回收，有更高优先级的线程需要运行，线程自己调用sleep等方法）
		当 Context Switch 发生时，需要由操作系统保存当前线程的状态，并恢复另一个线程的状态，Java 中对应的概念就是程序计数器（Program Counter Register），它的作用是记住下一条 jvm 指令的执行地址，是线程私有的
		***状态包括程序计数器、虚拟机栈中每个栈帧的信息，如局部变量、操作数栈、返回地址等
		***Context Switch 频繁发生会影响性能

--6.常用方法
	**start():启动一个新线程，在新的线程运行 run 方法中的代码(运行run方法前提是线程抢到了时间片)
	**run():新线程启动后会调用的方法
	**join():等待线程运行结束
	**set/getName():获取线程名
	**set/getPriority():优先级
	**getState()：获取线程状态
	**isInterrupted()：判断是否被打断
	**isAlive()：线程是否存活
	**interrupt()：打断线程（如果被打断线程正在 sleep，wait，join 会导致被打断的线程抛出 InterruptedException，并清除 打断标记；
		如果打断的正在运行的线程，则会设置 打断标记 ；
		park 的线程被打断，也会设置 打断标记）
	**interrupted()：判断当前线程是否被打断（会清除打断标记）
	**currentThread()：获取正在执行的线程
	**sleep():休眠当前线程
	**yield():提示线程调度器让出当前线程对CPU的使用

--7.主线程和守护线程
	默认情况下，Java 进程需要等待所有线程都运行结束，才会结束。
	有一种特殊的线程叫做守护线程，只要其它非守护线程运行结束了，即使守护线程的代码没有执行完，也会强制结束。
	**GC线程就是一种守护线程
	**Tomcat中的 Acceptor 和 Poller 线程都是守护线程，所以 Tomcat 接收到 shutdown 命令后，不会等待它们处理完当前请求
