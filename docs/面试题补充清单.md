## 三、后续还可以继续补的方向

如果还想把面试题库打得更完整，建议后续再补：

1. **系统设计与高并发场景题**：秒杀、订单、库存、优惠券、消息通知、审批流、租户隔离。
2. **可观测性与链路追踪**：日志规范、TraceId、Prometheus、Grafana、SkyWalking、慢接口定位。
3. **代码质量与工程规范**：DDD 分层、异常体系、参数校验、幂等、防重、灰度发布。
4. **定时任务与调度**：XXL-JOB、Quartz、分片任务、任务补偿、重复执行防护。
5. **文件存储与大文件处理**：MinIO、OSS、断点续传、预签名 URL、权限隔离。
6. **工作流 Flowable**：流程定义、任务流转、会签、驳回、动态表单和业务表绑定。

## 四、4 年 Java 面试复习优先级

建议按下面顺序刷：

```text
Java 基础
  -> 集合 / 并发 / JVM
  -> Spring / Spring Boot / MyBatis-Plus
  -> MySQL / Redis / MQ
  -> 微服务 / 网关 / 注册配置 / 限流熔断
  -> 安全权限 / 多租户 / 数据权限
  -> 分布式事务 / 分库分表
  -> 项目亮点和线上排查
```

4 年经验的面试重点不是“会不会用 API”，而是：

1. 能解释底层机制。
2. 能说出取舍和边界。
3. 能结合项目讲设计。
4. 能描述线上问题怎么排查和兜底。
